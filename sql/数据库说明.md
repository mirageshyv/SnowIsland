## 二、表结构详细说明

### 1. 用户与权限

#### user — 用户账户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | int(11) | PK, AUTO_INCREMENT | 用户ID |
| username | varchar(50) | NOT NULL, UNIQUE | 用户名 |
| password | varchar(100) | NOT NULL | 密码 |
| role | enum('dm','player') | NOT NULL | 角色：dm=主持人, player=玩家 |
| player_id | int(11) | FK → player.id | 关联玩家ID（DM角色为NULL） |
| status | tinyint(1) | NOT NULL, DEFAULT 1 | 账户状态（1=启用） |
| created_at | datetime | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | datetime | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引**：`username`(UNIQUE), `player_id`

## 四、业务逻辑说明

### 4.1 用户与权限

- `user` 表区分 DM（主持人）和 Player（玩家）两种角色
- DM 账户的 `player_id` 为 NULL，Player 账户通过 `player_id` 关联到具体角色
- DM 拥有全局操作权限（结算行动、管理天灾、管理仓库等）

### 4.2 物品体系（多态关联）

系统采用 **item_type + item_id** 的多态关联设计，四种物品类型共享同一套库存/交易/仓库表结构：

| item_type | 关联表 | 说明 |
|-----------|--------|------|
| item | item | 道具（医疗包、手电筒、钥匙等） |
| weapon | weapon | 武器（手枪、猎枪等，含 threat_level 威胁等级） |
| ammo | ammo | 弹药（手枪弹、猎枪弹等，关联 weapon 表） |
| material | material | 材料（木材、食物、燃料等） |

此设计应用于：`player_items`、`trade_items`、`shelter_stock`、`job_initial_items`、6张仓库库存表

### 4.3 仓库系统

- 6个仓库各自独立建表，结构完全相同
- `warehouse_config` 通过 `table_name` 字段动态映射到具体仓库表
- 每个仓库有对应的钥匙物品（`key_item_id` 关联 `item` 表），玩家需持有钥匙才能访问
- 搬运行动支持两种模式：仓库→仓库（上限500kg）、仓库→个人（上限300kg）

### 4.4 行动系统

玩家每天有2个行动槽位（action_slot: 1或2），支持7种行动类型：

| 行动类型 | 说明 | 结算方式 |
|----------|------|----------|
| go_location | 前往地点 | 自动结算（返回地点信息+NPC互动） |
| investigate_player | 调查玩家 | 批量结算（DM触发，返回目标玩家行动） |
| produce | 生产 | 批量结算（根据职业自动产出资源） |
| use_trait | 使用特性 | DM手动反馈 |
| use_skill | 使用职业技能 | DM手动反馈 |
| hide | 隐藏 | 自动结算（写入 player_stealth 表） |
| transport | 搬运 | DM结算（解析notes中的结构化数据执行搬运） |

- 搬运行动的 `notes` 字段存储结构化数据：`[mode:xxx]`, `[source:xxx]`, `[dest:xxx]`, `[item:type|id|qty|weight]`
- 隐藏行动写入 `player_stealth` 表，调查时检查目标是否在潜行状态

### 4.5 天灾系统

- `catastrophe_card` 定义11种天灾牌，每种3张副本（共33张）
- `catastrophe_deck` 管理牌组，每张牌有独立的抽取/使用状态
- DM 抽牌时从 deck 中随机抽取3张写入 `drawn_cards`
- 天灾使者从3张中选1张写入 `selected_catastrophe`
- `catastrophe_progress` 追踪天灾进度（0-100）
- `game_state` 管理全局游戏状态（天数、阶段、天灾触发等）

### 4.6 方舟系统

- `ark_construction` 为单例表（id=1），追踪方舟建造进度
- 建造需要投入木材、金属、密封材料，安装引擎/螺旋桨/发电机
- `ark_sail` 管理帆的建造（需要帆布+绳索+工作行动）
- `ark_voyage` 记录航海事件，航行天数由引擎数量和是否有帆决定
- `ark_config` 存储所有配置参数（目标量、转化率、惩罚等）

### 4.7 交易系统

- 玩家间交易采用请求-响应模式
- `trade` 记录交易主信息（发起方、接收方、状态）
- `trade_items` 记录每笔交易的物品明细，direction 区分 give（给予）和 take（索取）
- 交易状态流转：pending → accepted → completed / rejected / cancelled

### 4.8 反抗者里程碑

- 7个里程碑按 order_number 排序，需按顺序完成
- `milestone_player_status` 记录每个玩家对各里程碑的查看状态
- 反抗者阵营玩家可查看里程碑进度，DM 可标记完成
