## 4. 索引设计

### 4.1 道具表（item）
- PRIMARY KEY: id
- UNIQUE INDEX: name

### 4.2 武器表（weapon）
- PRIMARY KEY: id
- UNIQUE INDEX: name

### 4.3 子弹表（ammo）
- PRIMARY KEY: id
- UNIQUE INDEX: name
- FOREIGN KEY: weapon_id (关联weapon表)

### 4.4 物资表（material）
- PRIMARY KEY: id
- UNIQUE INDEX: name

### 4.5 玩家物资关联表（player_items）
- PRIMARY KEY: id
- FOREIGN KEY: player_id (关联player表)
- INDEX: player_id, item_type, item_id (组合索引，提高查询性能)
- INDEX: player_id, item_type (组合索引，用于按类型查询)

## 5. 数据查询优化

### 5.1 获取玩家拥有的道具
```sql
SELECT i.id, i.name, i.unit, i.remark, pi.quantity
FROM player_items pi
JOIN item i ON pi.item_id = i.id
WHERE pi.player_id = ? AND pi.item_type = 'item' AND pi.quantity > 0;
```

### 5.2 获取玩家拥有的武器
```sql
SELECT w.id, w.name, w.unit, w.remark, w.threat_level, pi.quantity
FROM player_items pi
JOIN weapon w ON pi.item_id = w.id
WHERE pi.player_id = ? AND pi.item_type = 'weapon' AND pi.quantity > 0;
```

### 5.3 获取玩家拥有的子弹
```sql
SELECT a.id, a.name, a.unit, a.remark, w.name AS weapon_name, pi.quantity
FROM player_items pi
JOIN ammo a ON pi.item_id = a.id
LEFT JOIN weapon w ON a.weapon_id = w.id
WHERE pi.player_id = ? AND pi.item_type = 'ammo' AND pi.quantity > 0;
```

### 5.4 获取玩家拥有的物资
```sql
SELECT m.id, m.name, m.unit, m.remark, pi.quantity
FROM player_items pi
JOIN material m ON pi.item_id = m.id
WHERE pi.player_id = ? AND pi.item_type = 'material' AND pi.quantity > 0;
```

## 6. 性能优化建议
1. **使用组合索引**：在player_items表上创建player_id、item_type、item_id的组合索引，提高查询效率
2. **添加数量过滤**：在查询时添加quantity > 0的条件，减少返回的数据量
3. **使用JOIN操作**：通过JOIN操作一次性获取关联数据，减少数据库查询次数
4. **考虑缓存**：对于频繁访问的玩家物资数据，可以考虑使用缓存机制
5. **批量操作**：对于批量添加或更新物资的操作，使用批量SQL语句减少数据库交互次数

## 7. 数据完整性保障
1. **外键约束**：确保player_items表中的player_id关联到player表的有效记录
2. **NOT NULL约束**：对必要字段添加NOT NULL约束，确保数据完整性
3. **UNIQUE约束**：对物资名称添加UNIQUE约束，避免重复数据
4. **默认值设置**：对数量等字段设置合理的默认值
5. **事务管理**：在进行物资操作时使用事务，确保数据一致性
